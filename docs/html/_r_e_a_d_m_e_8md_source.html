4 # Configurator Module
5 
6  Discover your network to found Upnp Rpc Devices
7  Special Devices
8  1. Homematic CCU
9  2. Enigma2 WebInterface
10  3. Fritzbox (see Add One Modules)

12 
13 Works:
14 
15 The search of devices in the network takes between 5 and 180 seconds,
16 depending on the timeout setting in the configurator and the number
17 of found device.
18 
19 The result is that the *local Windows* console seems to hang, but it is not like this :-(
20 In the *Webfront console*, you can track progress in the debug log.
21 
22 You can use **RpcConfigurator** to find and create new devices.
23 If your device does not appear in the list and you have the *device
24 description XML file URL* then you can either create a generic or multimedia
25 Module and enter the *full URL* as HOST. The module then tries to configure itself.
26 - NOTE When specifying multiple URLs, these are either separated as a string by comma or passed as a simple array
27 
28 - Example urls
29  - http://fritz.box:49000/tr64desc.xml
30  - http://192.168.112.61:8000/serverxml.xml,http://192.168.112.61:8080/description.xml
31  - homematic.xml for CCU
32  - enigma2.xml for DreamBox Enigma2 WebInterface
33 
34 All modules include timer functions for updating the status variables, with the exception of the RPCGENERIC module.
35 There are 2 timer modes *UPDATE* and *OFFLINE* which can each have separate values, this means when the device is *online*
36 the status is updated every 15min, if it is *offline* (TV off) the check / update will take place only every 2 hours
37 until the device is *online* again.

41 - **xxx_GetApiInfo**(IpsInstanceID) Returns an array of informatons to the API

44 # Generic RPC Module
45  API to call all methods discovered from device only one Variable for Status created
46 
47 This module also exports the function
48 - **RPCGENERIC_CallMethod**(IpsInstanceID, FunctionName, comma-separated string with parameters)
49 
50 Works:
51 
52 the syntax is the same as **$api**-> __ call (FunctionName, Parameter), with the difference that the parameters are passed not as an array but as a comma-separated string.
53 - **RPCGENERIC_CallApi**(IpsInstanceID, 'SetVolume', "0, Master, 10")
54 - **RPCGENERIC_CallApi**(IpsInstanceID, 'GetVolume', "0, Master")
55 or
56 - **$api** = **RPCGENERIC_GetApi**(IpsInstanceID)
57 - $volume = **$api**-> GetVolume (0, "Master")
58 - **$api**->SetVolume (0, "Master", $ volume)
59 
60 Calling it via the *RpcApi* object **$api** is useful in any case when calling several, consecutive, calls, as it does not make sense, as with xxx_GetApi (IpsInstanceID), with each call the module is recreated.
61 
62 Furthermore, it is possible to address calls directly to an service. Since some Rpc devices such
63 as the Fritzbox contains several functions of the same name, GetInfo(), it is
64 necessary to transfer the service name. This happens as follows
65 - **$api**->__ call ("DeviceInfo1.GetInfo", array with parameter)
66 or
67 - **$api**->{"DeviceInfo1.GetInfo"}(parameter,parameter...)

70 # Multimedia RPC Module
71  Includes Standard methods for
72  1. Volume, Mute (Generic)
73  2. Bass, Loudness, Trebble (sonos)
74  3. Play,Pause,Stop,Next,Previous (Generic)
75  4. Color,Brightness,Sharpness, Contrast (TV )
76 
77 This module also exports the functions
78 - **RPCMEDIA_RequestUpdate**(IpsInstanceID) updates all status variables
79 - **RPCMEDIA_WriteValue**(IpsInstanceID,statusvar,value) set status variable
80 
81 Works:
82 
83 Since InstanceID is usually 0, this variable does not have to be specified, just
84 like Channel, these values are automatically added when called.
85 
86 Therefore the call with **$api**->GetVolume() or **$api**->SetVolume(10) is also possible.
87 
88 The volume can be changed or read as follows
89 - **$api**=**RPCMEDIA_GetApi**(IpsInstanceID)
90 - $volume=**$api**->GetVolume()
91 - **$api**->SetVolume(NewVolume)
92 
93 or use the **RPCMEDIA_WriteValue** function to change status variables.
94 - **RPCMEDIA_WriteValue**IpsInstanceID, 'VOLUME', 10) sets the volume to 10
95 - **RPCMEDIA_WriteValue**(IpsInstanceID, 'PLAYSTATE', value) is used to control the playback, whereby the following values are possible:
96  - 0: Stop
97  - 1: Pause
98  - 2: Play
99  - 3: Next
100  - 4: Prevoius
